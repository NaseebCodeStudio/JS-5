• Concatenation in JavaScript refers to the process of joining two or more strings together to create a new string. It is commonly achieved using the + operator or the concat() method.
    Here's how you can concatenate strings using the + operator:
    const firstName = "Naseeb";
    const lastName = "Khan";
    const fullName = firstName + " " + lastName;
    console.log(fullName); // Output: "Naseeb Khan"

    You can also use template literals
    Template literals "backticks ``", introduced in ES6/ES2015, are a way to create strings in JavaScript that allow for easier string interpolation and multiline strings:
    const firstName = "Naseeb";
    const lastName = "Khan";
    const fullName = `${firstName} ${lastName}`;
    console.log(fullName); // Output: "Naseeb Khan"
    In this example, the variables names are included directly within the string using ${}. When the code runs, the values of names are automatically inserted into the string.
    Template literals also support multiline strings, which means you can include line breaks within the string without using special escape characters like \n.
    const multiline = `
    This is a
    multiline
    string.
    `;
    console.log(multiline);
    This makes template literals a convenient and more readable way to work with strings in JavaScript.
    Now, imagine you have some information that you want to display neatly, like in a table. console.table() in JavaScript lets you do just that.
    For example, if you have a list of items with properties like name, age, and country, you can display them in a table format in the console using console.table([]).
    const name1 = "Naseeb";
    const name2 = "Khan";
    const age = 22;
    console.table([name1, name2, age]);
    When you run this code, you'll see the information displayed in a neat table format in the console, making it easier to read and understand, especially for larger datasets.

  • - Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their containing scope during the compilation phase, before the code is executed.
    - In simpler terms, variables and functions in JavaScript can be used before they are declared. When JavaScript code is executed, variable and function declarations are "hoisted" to the top of their containing scope. However, only the declarations are hoisted, not the initializations or assignments.
    For Example:
    console.log(x); // Output: undefined
    var x = 5;
    In the above code, even though x is logged before it's declared and assigned a value, it doesn't cause an error. This is because the declaration of x is hoisted to the top of the scope by the JavaScript engine during compilation. However, the value is not hoisted, so at the time of logging, x is undefined.
    Similarly, function declarations are also hoisted.

  • In JavaScript, reference data types (also known as reference types) are types that hold references to objects stored in memory. Unlike primitive data types (such as numbers, strings, booleans, null, and undefined), reference data types do not directly contain their values; instead, they contain references (memory addresses) to the locations in memory where the actual data is stored.
    The main reference data types in JavaScript are:

    Object: Objects are collections of key-value pairs, where keys are strings (or symbols) and values can be of any data type, including other objects, functions, arrays, and primitives.
    const person = {
    name: "Naseeb Khan",
    age: 22,
    city: "Quetta"
    };
    Array: Arrays are ordered collections of elements, each identified by an index. They can hold elements of any data type, including other arrays and objects.
    const numbers = [1, 2, 3, 4, 5];
    Function: Functions are objects in JavaScript. They can be assigned to variables, passed as arguments to other functions, and returned from functions. Functions can also have properties and methods.
    function greet(name) {
    console.log("Hello, " + name + "!");
    }
    Date: The Date object represents a specific point in time. It allows you to work with dates and times, including creating, formatting, and manipulating dates.
    const currentDate = new Date();
    RegExp: The RegExp object represents a regular expression pattern. It is used for pattern matching and searching within strings.
    const pattern = /[a-z]+/i;
    Symbol (Introduced in ES6/ES2015): Symbols are unique and immutable primitive values that can be used as object property keys to avoid naming collisions.
    const mySymbol = Symbol();
    Reference data types are stored and manipulated by reference, meaning that when you assign a reference data type to a variable or pass it as an argument to a function, you're actually working with a reference to the original data stored in memory, rather than a copy of the data itself. This has implications for how reference data types behave in terms of assignment, comparison, and function calls.

  • In JavaScript, an object is a complex data type that allows you to store collections of key-value pairs, where keys are strings (or symbols) and values can be of any data type, including other objects, functions, arrays, and primitives.
    Objects in JavaScript are similar to real-life objects in that they have properties and methods. Properties are the characteristics or attributes of an object, while methods are the functions associated with the object that can perform actions or calculations.
    Here's how you can create and work with objects in JavaScript:

    Object Literal:
    You can create an object using object literal notation, which consists of curly braces {}.
    const person = {
    name: "Naseeb Khan",
    age: 22,
    city: "Quetta",
    greet: function() {
    console.log("Hello, " + this.name + "!");
    }
    };
    Accessing Properties:
    You can access object properties using dot notation (objectName.propertyName) or bracket notation (objectName['propertyName']).
    console.log(person.name); // Output: "Naseeb Khan"
    console.log(person['age']); // Output: 22
    Adding and Modifying Properties:
    You can add new properties to an object or modify existing properties.
    person.job = "Engineer";
    person['age'] = 25;
    Methods:
    Objects can also contain functions, known as methods, which can be used to perform actions related to the object.
    person.greet(); // Output: "Hello, Naseeb Khan!"
    Object Constructor:
    You can create objects using constructor functions and the new keyword.
    function Person(name, age) {
    this.name = name;
    this.age = age;
    this.greet = function() {
    console.log("Hello, " + this.name + "!");
    };
    }
    const john = new Person("Naseeb Khan", 25);

  • 1. Dot Notation:
    Dot notation is a way to access properties and methods of an object using a dot (.) followed by the property or method name. It's the most common and straightforward way to access object properties.
    const person = {
    name: "Naseeb Khan",
    age: 23
    };
    console.log(person.name); // Output: "Naseeb Khan"
    console.log(person.age); // Output: 23

    2. Bracket Notation:
    Bracket notation is another way to access object properties and methods using square brackets []. It allows you to access properties dynamically, using variables or expressions inside the brackets.
    const person = {
    name: "Naseeb Khan",
    age: 23
    };
    console.log(person['name']); // Output: "Naseeb Khan"
    console.log(person['age']); // Output: 23
    You can also use bracket notation to access properties with special characters or reserved words:
    const person = {
    'first name': "Naseeb",
    'last name': "Khan"
    };
    console.log(person['first name']); // Output: "Naseeb"
    console.log(person['last name']); // Output: "Khan"
    When to Use:
    - Use dot notation when you know the property name in advance and it's a valid identifier.
    - Use bracket notation when the property name is dynamic (e.g., stored in a variable) or contains special characters.
    Both dot notation and bracket notation can be used interchangeably in most cases, but bracket notation provides more flexibility when working with dynamic property names or special characters.

  • Imagine you have a bunch of items that you want to keep together in a list. An array in JavaScript is like a container where you can store multiple pieces of information, such as numbers, strings, or even other arrays, all in one place.
    Here's how you can create an array:
    const fruits = ["apple", "banana", "orange", "kiwi"];
    In this example, fruits is an array that contains four elements: "apple", "banana", "orange", and "kiwi".
    You can access individual elements in the array using their index. In JavaScript, array indexes start from 0. So, to access the first element "apple", you would use fruits[0].
    console.log(fruits[0]); // Output: "apple"
    console.log(fruits[1]); // Output: "banana"

    Arrays are versatile and powerful data structures in JavaScript, allowing you to store and manipulate collections of data efficiently. They are used extensively in programming for tasks like storing lists of items, managing data, and performing operations on groups of elements.

  • - In JavaScript, functions are blocks of code that perform a specific task. They are like small, reusable programs within your code.
    - Functions take inputs, perform operations on them, and can return a result. They help to organize your code, make it more readable, and enable you to reuse code instead of writing the same instructions multiple times. You can define functions using the function keyword followed by a name, parameters (if any), and the code block enclosed in curly braces {}.
    Here's an example:
    1.
    function greet(){
    console.log("hello world!");
    }
    // calling:
    greet();

    OR
    2.
    // Defining a function named 'greet'
    function greet(name) {
    return "Hello, " + name + "!";
    }
    // Calling the 'greet' function
    console.log(greet("Naseeb")); // Output: "Hello, Naseeb!"
    - In this example, greet is a function that takes one parameter name. Inside the function, it concatenates the string "Hello, " with the provided name and returns the result. When we call the greet function with the argument "Naseeb", it returns "Hello, Naseeb!", which we then log to the console. Functions are essential building blocks in JavaScript programming, enabling you to write modular and reusable code.

  • In JavaScript, functions can have parameters and accept arguments. Here's a breakdown of the difference between arguments and parameters:
    Parameters:
    - Parameters are placeholders defined in the function declaration.
    - They are like variables that represent the values that the function expects to receive when it is called.
    - Parameters are listed inside the parentheses of the function declaration.
    - Parameters are used to define the function's behavior and what kind of input it can accept.
    Example:
    function greet(name) {
    return "Hello, " + name + "!";
    }

    In this example, name is a parameter of the greet function.
    Arguments:
    - Arguments are the actual values passed to the function when it is called.
    - They are the concrete values that replace the parameters when the function is executed.
    - Arguments are provided inside the parentheses when calling the function.
    Example:
    console.log(greet("Naseeb"));
    n this example, "Naseeb" is an argument passed to the greet function.
    In summary, parameters are the placeholders defined in the function declaration, while arguments are the actual values passed to the function when it is called. Parameters are used to define a function's behavior, while arguments are the data that the function operates on.
